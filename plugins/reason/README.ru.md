# Плагин Reason

Гипотезо-ориентированное рассуждение с аудируемым следом доказательств, основанное на [First Principles Framework (FPF)](https://github.com/ailev/FPF).

## Команда

### `/reason` — цикл рассуждения ADI

Проводит решение через полный цикл Abduction-Deduction-Induction:

1. **Abduction** — генерирует 3-5 конкурирующих гипотез с разных ракурсов
2. **Deduction** — логически верифицирует каждую гипотезу на соответствие ограничениям
3. **Induction** — собирает эмпирические доказательства с оценкой надёжности и релевантности
4. **Decision** — ранжирует гипотезы по trust score и создаёт Design Rationale Record

### Когда использовать

- Архитектурные решения: «монолит или микросервисы?»
- Выбор технологий: «какая БД для нового сервиса?»
- Анализ компромиссов: «скорость или корректность в этом пайплайне?»
- Любой вопрос, где важно «почему это, а не то»

### Когда НЕ использовать

- Быстрые вопросы — просто спросите
- Реализация — используйте `/build`
- Экспертный анализ без требований к доказательствам — используйте `/think`
- Экспертные дебаты — используйте `/arena`

## Агенты

| Агент | Роль | Жизненный цикл |
|-------|------|----------------|
| `hypothesizer` | Генерирует одну обоснованную гипотезу с определённого ракурса | одноразовый |
| `verifier` | Логически верифицирует гипотезу (согласованность, ограничения, режимы отказа) | одноразовый |
| `evidence-gatherer` | Собирает эмпирические доказательства и оценивает по R (надёжность) и CL (релевантность) | одноразовый |

## Ключевые концепции

- **WLNK (Weakest Link)** — trust = min(оценок доказательств), не среднее
- **Уровни гарантии** — L0 (наблюдение) → L1 (обосновано) → L2 (верифицировано)
- **DRR (Design Rationale Record)** — аудируемый артефакт, сохраняется в `docs/decisions/`
- **Transformer Mandate** — AI рекомендует, человек решает

## Результат

Design Rationale Record сохраняется в `docs/decisions/YYYY-MM-DD-[topic]-drr.md`
