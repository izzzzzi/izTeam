# Think Plugin

Структурированное планирование через параллельных экспертов. Версия: 1.1.7.

## Скилл: /think

**Аргумент:** `<задача или идея для обдумывания>`

### Стадии

#### Stage 1: Task Breakdown

1. Понять задачу (1-2 предложения)
2. Выбрать главного эксперта + принципы от 3 экспертов
3. Разбить на 5-15 аспектов (max 15)
4. Назначить эксперта каждому аспекту

**Output:** таблица аспектов с экспертами

#### Stage 2: Project Study

Запуск `think:expert` агентов параллельно — один на аспект, в одном сообщении.

**Decision tree:**
- 0 аспектов → задача слишком простая
- 1-2 → запуск + предупреждение
- 3-15 → стандартный путь
- >15 → группировка связанных аспектов

**Error handling:**
- <50% экспертов вернулись → partial results + warning
- 0 вернулось → report failure

#### Stage 3: Summary Document

Создание unified документа и сохранение в `docs/plans/YYYY-MM-DD-[topic]-design.md`.

**Структура документа:**
1. Overview (Goals + Key Decisions table)
2. Детальный анализ каждого аспекта (эксперты, решение, код)
3. Implementation Plan (фазы с чеклистами)
4. Success Metrics (таблица baseline → target)

Диаграммы: mermaid `flowchart TD` (не ASCII art).

## Агенты

### think:expert

Канонический эксперт для анализа одного аспекта:
- Полное изучение проекта (Read, Glob, Grep, Bash)
- Предложение вариантов решения
- One-shot агент
